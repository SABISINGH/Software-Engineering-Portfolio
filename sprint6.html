Technical achievment

As I mentioned in the professional achievement, I think I have achieved a lot of technical achievements in this sprint. Some of the videos I watched on YouTube were not even in English or a language I could understand Link but I manage to learn and implement from them to my project. This website was very helpful to understand what was happening behind the scenes and also the front end. Now I will write what I have implemented/added to my admin user control. Firstly, I started with role creation by setting up roles table and model by making model with -m for migration as well and Linked users to Roll. Created migration between role & user. Created RolesTableSeeder and inside the table I applied truncate method tot he table so this will delete everything in the table every time the seed in done for best practice as we never want the duplication of the data. Created UserstableSeeder for users, added roles. Created 2 users For the roles and attached the roles method to the users using the attach method for my testing purposes. Created helper method on the User model to help the roles of the users an check if they have a certain role. Created new middleware to test if the user is admin and only admin can see the users and added auth.admin middleware for create-admin-interface-laravel Knows about the middleware and the routes can be protected by the specific role. I used my last year knowledge from web2 and created a Admin\\UseController resource controller. A new part I learnt this time was we can actually delete some of the functions and then let larval know that we have deleted functions through routes. While I was creating the index page to show the users under manage users I learnt the onclick button event to actually ask the user again if they want to go ahead with the request and implemented the Heuristics for User Interface Design points. I have mentioned this in my professional achievements. While I was working through Use Controller and creating the editing method another very interesting part I learnt was when we redirecting the process of edit/update/delete we can actually pass the message to the user using the alerts of different colors. I have also learnt how to add the alerts from Bootstraps alerts site. In my destroy method I created a check for admin not allowed to edit/delete to avoid admin removing their own admin rights and lock them out of the access. I have also learnt how we can create fake users by seeding so we can practice Managing them through my admin role from ‘faker’ and from laravel site and by doing so I saved a lot of time on doing my testing of the account. I have learnt how to use larval factory. Larval comes with a factory created in database/factories/userfactory. It doesn't attach any roles to the factory so I used afterCreating method and attached roles to it. I have learnt both ways of creating users with tinker process in the gitbah using "php artisan tinker" command and then "factory(App\user::class, 5)->create()" method. Another method was to call our factory in the useTableSeeder. After truncating the user table I have added "DB::table('role_user')->truncate();"
I have realized that our app can have unlimited users and having them displayed on one page could be an issue, so I wanted to add pagination to add that extra feature. I have never tried pagination before, but when I searched online for laravel.com I learnt that it wasn't that hard. Inside my use Controller instead of having all I added paginate(10), allowing 10 users per page and just let the view now about the change by adding '$users ->links()' just after the table end. I always thought the pagination is quite hard to implement, but while I was working on my site.
